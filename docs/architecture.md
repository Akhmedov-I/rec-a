# Архитектура и Техническое Задание проекта Rec-A (Recruiter Assistant)

## 1. Роли и Права Доступа (Role-based Access Control)
Платформа поддерживает два типа регистраций: **Компания** и **Частный Рекрутер**.

### Тип: Компания
Первый зарегистрировавшийся пользователь компании получает роль **Админ**.
- **Админ (Admin):** Управление всеми аккаунтами компании, приглашение сотрудников, назначение ролей.
- **Заявитель (Requester):** Создает заявки на подбор. Видит только свои заявки и их статусы.
- **Рекрутер (Recruiter):** Принимает и обрабатывает заявки, работает с кандидатами, проводит тестирования.
- **Руководитель (Manager):** Видит все отчеты по подбору, статусы, может создавать заявки.
- **HRD:** Получает все новые заявки, назначает их на конкретных рекрутеров, имеет доступ к отчетам как Руководитель.
*(Любая роль может создать заявку на подбор)*

### Тип: Частный Рекрутер
- **Частный рекрутер (Private Recruiter):** Единая роль без иерархии. Сам создает заявки, сам обрабатывает, имеет доступ ко всем функциям и отчетам в рамках своего аккаунта.

## 2. Основные Модули и AI-Интеграция

### 2.1. Управление Заявками (Requisitions)
- **Создание:** Загрузка файла (PDF, Word, Excel, Скриншот) или ввод нескольких строк текста.
- **AI Обработка (Gemini):** Извлекает и структурирует требования: образование, опыт, сфера, личные навыки, психотип, обязанности, условия.
- **Редактирование:** Пользователь может изменять или дополнять сгенерированные AI данные.

### 2.2. Воронка Кандидатов (Candidates Pipeline)
- **Добавление:** Рекрутер прикрепляет резюме к заявке.
- **AI Анализ (Gemini):** Глубокий анализ резюме на соответствие требованиям вакансии. Оценка психотипа по тексту. Выдача рейтинга от 0 до 100 баллов и подробного отчета.
- **История:** Сохранение всех действий (ERP для рекрутинга).
- **Статусы:** Новые -> Тестирование -> Собеседование -> Оффер -> Принят/Отказ.

### 2.3. Модуль Тестирования (Testing System)
- **Инициация:** Рекрутер выбирает кандидата, указывает необходимые компетенции для проверки.
- **Ссылка:** Система (AI) генерирует одноразовую уникальную ссылку с инструкцией. Если ссылка испорчена — можно сгенерировать новую.
- **Прохождение теста (Кандидат):**
  - Защита от копирования текста (отключение ПКМ, выделения).
  - Ввод данных: ФИО, месяц и год рождения.
  - Блоки по 10 минут (максимум 6 блоков).
    1. **Психологический портрет (По умолчанию):** Определение психотипа и соответствия должности.
    2. **Логика (По умолчанию):** Проверка скорости мышления, от простого к сложному.
    3-6. **Дополнительные компетенции:** Задаются рекрутером (например, знание React).
  - Вопросы генерируются AI адекватно уровню, но с жестким лимитом времени, чтобы исключить поиск в Google/AI.
- **Результат:** Автоматический отчет с рекомендациями от AI для рекрутера, доступный для красивой печати (PDF).

### 2.4. Собеседование и Оффер
- **Оффер:** Встроенный конструктор офферов с помощью AI.
- **Закрытие:** Принятие оффера закрывает заявку. Отказы сохраняются в истории кандидата.

## 3. Стек Технологий и Инфраструктура
- **Фреймворк:** Next.js (App Router, React 18+).
- **База Данных & Auth & Storage:** Firebase (Firestore, Firebase Authentication, Cloud Storage).
- **Локализация:** `next-intl` (Русский - основной, Узбекский).
- **Стилизация:** Tailwind CSS, `lucide-react` (иконки).
- **ИИ-Провайдер:** `@google/generative-ai` (Gemini 1.5 Pro/Flash).
- **Парсинг файлов:** `pdf-parse` (PDF), `mammoth` (Word/DOCX).

---

## 4. Структура Базы Данных (Firestore)
- `users`: Хранение информации о пользователях (email, role, companyId).
- `companies`: Данные о компаниях.
- `requisitions`: Заявки на подбор (title, description, requirements, aiGenerated, status).
- `candidates`: Кандидаты, привязанные к заявкам (resumeUrl, aiRating, aiAnalysis, status).
- `tests`: Сгенерированные тесты (candidateId, token, blocks, status, results).
- `offers`: Сгенерированные предложения о работе.
